# AGENTS.md

## 코딩 전에 생각하기

- 가정을 명확히 설명할 것
- 불확실하면 추측하지 말고 질문할 것
- 가능하면 더 단순한 방법과 트레이드오프를 함께 제시할 것

## 단순함 우선

- 필요한 최소한의 코드만 작성할 것
- 불필요한 추상화/설정화 금지
- 필요 없는 에러 처리 추가 금지
- 복잡한 코드보다 읽기 쉬운 코드를 우선할 것

## 필요한 부분만 수정하기

- 요청된 부분만 수정할 것
- 기존 코드 스타일을 따를 것
- 관련 없는 리팩토링 금지
- 자신의 수정으로 생긴 불필요한 코드만 제거할 것
- 
## 일반 원칙

- 애매한 상황에서 임의로 결정하지 말 것
- 불확실성은 초기에 드러낼 것
- 모든 수정은 요청과 직접 연결되어야 함
- 특별한 이유가 없다면 더 단순한 해결책을 선택할 것
